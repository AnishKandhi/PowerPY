Classes & Objects: Declarations

Think of classes as blueprints. They are used as templates to build specific models. In programming, these specific "instances" built are called objects. For example, a car class may provide the blueprint/structure of the car, while the object is a specific car built from that blueprint to be sold. The class sets the attributes/methods, including the car's color, name, and mileage, while the object is a unique combination of those attributes and contains the methods, which are the actions/uses of the car (eg. driving, breaking). Classes are defined by the keyword 'class' followed by the class name and a semicolon, with subsequent indentation.

Constructors

Constructors are essentially what helps build the object. They provide the necessary attributes that the object must be defined with when an object is instantiated. For example, the constructor for a car class would include an object's state variables/attributes, like color, name, and mileage. Contructors are defined using def __init__(), which has two underscores on each side. An object is instantiated by setting the object name to the class name, parentheses, and the contructor parameters in the parantheses. See snippet for clear breakdown: 


class Car:
    def __init__(self, name, color): #constructor
        self.name = name  #initializing attributes
        self.color = color
        

#-------Object Instanitation------
car = Car("BMW", "blue") 
#must match constructor
print(car.name) #BMW
print(car.color) #blue

Methods

Methods are simply functions defined in and as part of a class. So a method is called through an object of a class, since the methods belong to the object, unless declared static. Here is how methods are defined and called. 


class Car:
    def __init__(self, name, color): 
        self.name = name 
        self.color = color

    def getColor(self):  #method
        return self.color

    def getName(self):  #method
        return self.name
        

----Calling Methods-----
car = Car("BMW", "blue") 

print(f"The car is a {car.getColor()} {car.getName()}")
#This car is a blue BMW

Static Methods

While regular methods belong to the object, static methods belong to the class. This means that in order to call a static method, we much use the class name to do so, rather than creating an object. A method is static when the 'self' keyword is not put into the paramters, indicating the method does not belong to the object. A @staticmethod decorator is used over the method to tell python the method is static. 


class Car:
    def __init__(self, name, color): 
        self.name = name 
        self.color = color
    
    @staticmethod
    def getColor():  #static method
        return "blue"
    
    @staticmethod
    def getName():  #static method
        return "BMW"
        

------Calling Method------
#No object needed, use class name
print(f"This car is a {Car.getColor()} {Car.getName()}")
#This car is a blue BMW

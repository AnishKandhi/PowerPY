User Input Handling in Python

So far, we have been using the print() function, which outputs values to the developer console. With python, it is also very simple to prompt and capture user input, using the input() function. The code segment to the right asks the user to input their name, stores the string in a variable name, and outputs the name.


name = input("Enter your name: ")
print(name)

Input Function Details

The input function asks the user whatever string is in its parameter. It then returns the user's input and stores it in the variable the input is assigned to. The input function only captures the user input as a string. So if you want to extract an int/float value from the user, you have to cast their input by using the int() or float() functions.


num1 = input("Enter number one: ") 
#user enters 5
num2 = input("Enter number two: ") 
#user enters 4
print(num1 + num2) #54 -- strings are concatenated
- - - - - - - - - - - - - - - - - - - - - - - - - -
num1 = int(input("Enter number one: ")) 
#user enters 5
num2 = int(input("Enter number two: ")) 
#user enters 4
print(num1 + num2) #9 -- integers are added

Exception Handling: ValueError

When we try to cast a string from an input into an int or float, some problems may arise. What if the user enters a word or character that can not be casted to a number? Here, we need to catch that exception early so it does not break the program. For that, we use a 'try' & 'except' block, where the 'try' block attempts to excecute a risky line of code, and the except block catches the exception and handles it by running alternate code. Example below:


num1 = int(input("Enter number one: ")) 
#user enters "five" -> ValueError 
#String can't be converted to int
- - - - - - - - - - - - - - - - - 
try:
    num1 = int(input("Enter number one: ")) 
except:
    print("Invalid number entered. . .")
#the code tries to cast the user's input to an int
#if it can't be casted, the except block code runs

Exception Handling: Arithmetic

What if the values the user entered can not be computed mathematically? For example, the values try to divide by 0. This raises an arithmetic exception error. A try/except block can handle this as well.


num1 = int(input("Enter number one: ")) 
#enters 10
num2 = int(input("Enter number two: ")) 
#enters 0
print(num1/num2) #ZeroDivisionError
- - - - - - - - - - - - - - - - - 
try:
    num1 = int(input("Enter number one: ")) 
    #enters 10
    num2 = int(input("Enter number two: ")) 
    #enters 0
    print(num1/num2)
except:
    print("Can not divide by zero")

Handling Multiple Exceptions

Sometimes there are multiple possible excpetions that may arise. In the example above, there could have been a value error, if the user entered an non-castable value, as long as a zero division error. To handle this, we need to be more specific in our except blocks. See code snippet for reference.


try:
    num1 = int(input("Enter number one: ")) 
    #enters 10
    num2 = int(input("Enter number two: ")) 
    #enters 0
    print(num1/num2)
except ZeroDivisonError:
    print("Can not divide by zero")
except ValueError:
    print("Invalid input: Enter numerical value")
else:
    print("Successfully computed!")
#The else statement runs if no exception caught
